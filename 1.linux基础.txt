7.	命令
		什么是命令？
			由于用户是需要和系统交互.
			操作系统提供自身的可执行的程序..这种程序叫命令.
			后面我们自己写shell的时候..就是脚本命令.你可以这样理解.

		命令的格式
			命令   -选项(选项是可以合并的)   -参数(接收对象)
			example: ls -hal(选项) /etc（参数） 
			cd /etc(参数) 

	ls命令 列出路径文件 （默认是current路径）
		文件的类型：第一个字母
			- 	普通文件
			d	目录文件

			b	块设备文件
			l   符号链接文件（symbolic link file）
			c 	字符设备文件（）
			p   管道文件
			s	套接字文件（socket）

		文件权限
			文件的属主：就是创建文件的用户

				r read  可读,  可以打开
				w write 可写  可以写入,就是你可以更改文件
				x       可执行 

			文件的属组：就是创建文件的用户，我和你是一个组的。
				r
				-   没有写的权限
				x
			随便什么人都可以有的权限
				-   没有读的权限
				-   没有写的权限
				-   没有执行的权限

			d     rwx  r-x  ---.  7（这个是被硬链接的次数，明天再讲） 
			root（用户名） root（组的名字）     
			4096（文件大小） Oct 15 23:19 firewall

	
	cd   changed directory 改变当前所在的目录

		显示出当前所在的路径

	clear  清除当前屏幕信息

	history 显示出历史命令，就是你前500个命令

第二天.========================================================
文件路径：
linux下有一个标准协议。在根目录下的某一些目录都是固定的。不管是哪个发行版本。

1. linux standard base 
	就是说固定的几个目录在根目录下不论是什么发行版本都是存在的
	比如说： bin , home , var ,usr.这些目录是一定会存在的。


2. linux的哲学思想。
	1. 一切都是文件.
	2. 由众多的小程序来实现一个大功能.
	3. 所有配置都是由纯文本来保存
		你只要学一个vim编辑器就好了.
	4. 尽量避免跟用户交互

3. 文件属性：有两种
	元数据 metadata ： 文件名，文件大小。文件创建时间，是不是需要一个地方存储这些信息 
	数据  data      ： 文件的数据，是可以改变文件的大小

4. linux的创建文件的要求
	1. 文件名严格区分大小写
	2. 文件名可以使用任意的字符，但是不包括"/" 为什么呢?
		因为/用于划分文件路径和它本来就是根目录
	3. 所有的以.开头的文件名,都是隐藏文件	


5. 文件路径
	绝对路径： 就是从根目录开始的
	相对路径： 对于当前位置起始的路径
	当前路径： ./ 表示当前路径


6.	获取命令帮助
	命令是有分：外部命令和内部命令。
		外部命令就是：发行商写好的。
		内部命令就是：linux自带的

	内部命令：help [COMMAND] 
	外部命令：
		COMMAND --help
		info COMMAND
		万能的man：command 获取命令手册
			J:下一行
			k:上一行
			q:退出man的shell
